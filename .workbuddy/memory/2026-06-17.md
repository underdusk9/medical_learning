## 笔记框功能（题目页面底部显示笔记）

在答题页面题目卡片底部添加笔记显示框，支持两种模式：
1. **一直显示**（默认，`showNoteWithAnswer=false`）
2. **答题后显示**（`showNoteWithAnswer=true`）

### 改动文件

- **lib/widgets/question_card.dart**：
  - 新增 `noteContent`（String?）和 `showNoteWithAnswer`（bool，默认 false）参数
  - 解析框下方新增笔记框：浅黄背景（#FFF8E1）、`Icons.note_outlined` + "笔记"标题、AppColors.bookmark 配色
  - 显示条件：`noteContent != null && !empty && (!showNoteWithAnswer || isAnswered)`

- **lib/screens/quiz_screen.dart**：
  - 导入 `note_provider.dart`
  - 新增 `ref.watch(noteByQuestionProvider(question.id))` 监听当前题目的笔记
  - 将 note content 传递给 `QuestionCard`

## 自定义启动图（彩蛋功能）

在设置页新增自定义启动图功能（密码保护彩蛋）。

### 改动文件

- **pubspec.yaml**：注册 `assets/images/` 目录
- **assets/images/custom_splash.jpg**：用户放入的自定义启动图
- **lib/core/constants.dart**：新增 `prefCustomSplashEnabled` 和 `prefCustomSplashUnlocked` 键
- **lib/providers/settings_provider.dart**：新增 `customSplashProvider`、`customSplashUnlockedProvider`（StateProvider<bool>）、`verifyAndToggleSplash()` 密码验证+解锁+切换、`toggleSplash()` 已解锁后直接切换。`initSettings` 统一初始化所有设置。
- **lib/providers/init_provider.dart**：调用 `initSettings` 替代单独的 `initNoteDisplayMode`
- **lib/screens/settings_screen.dart**：新增自定义启动图 SwitchListTile。`_handleSplashToggle()` 先检查解锁状态，未解锁弹密码框，已解锁直接调 `toggleSplash()`
- **lib/main.dart**：改为 ConsumerStatefulWidget，手动控制启动流程。`_startLoading()` 等待 initProvider 完成，若 splashEnabled 则至少延迟 2 秒再切换。解决 Riverpod 缓存导致 loading 分支不显示的 Bug。

### 交互流程
1. 设置页 → 点击"自定义启动图"开关
2. 弹出密码输入框 → 密码正确则切换开关
3. 密码错误 → SnackBar 提示"密码错误"
4. 首次验证通过后后续可直接切换
